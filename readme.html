TEIL 2: TECHNISCHE DOKUMENTATION FÜR ENTWICKLER (Pine Script v6)

1. Architektur und Ausführungslogik

Das Pine Script ist strikt modular aufgebaut. UI ist in Tabellen (nun 22 Zeilen) gekapselt und Alarme reagieren freq_once_per_bar bei harten Bestätigungen (barstate.isconfirmed). Das neue v20.5 HUD greift nahtlos auf diese Logic-Engine zu.

2. Dynamisches Risiko & ATR Hysterese

  • atr_buffer = atr_val * 0.2 verhindert Fakeouts an den Bollinger Bändern durch einen Noise-Puffer. Dadurch schwankt das System im Radar-Mode signifikant weniger.
  • use_dyn_risk schaltet den VIX-Quotienten ein: risk_per_trade * (20.0 / vix_c). Das schützt in Panikphasen das Kapital durch intelligente Positionsverkleinerung und berechnet shares dynamisch.

3. Lower Timeframe Repaint-Schutz (RLD)

Anstatt nur per Standard-Request zu pushen, fragt v20.5 die Lower TF Daten hart ab: request.security_lower_tf(syminfo.tickerid, "240", volume).

  • Die Relative Liquidity Density (RLD) rechnet: (current_bar_lower_vol / avg_anchor_vol) * 100.
  • Ein Volumen-Vakuum bis zum TP (Take Profit) wird nur freigegeben (rld_valid = true), wenn der Pct-Wert unter rld_limit (20.0%) liegt. Das bedeutet echte Beschleunigungs-Leere (Thin Air).

4. Anchored Iterator

Die eigene MTF Volumen-Funktion f_calc_tf_vol_anchored bekommt einen dynamischen Lookback in anchor_bars übergeben. Dadurch wird die Density-Ratio nicht durch irrelevante alte Historie der Aktie oder simple Moving Averages vor dem letzten Trendbruch verzerrt. Es fängt genau dort an, wo das institutionelle Geld den Wechsel eingeleitet hat.
