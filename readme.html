II. Das Asymmetrische Risiko-Regime

Die „Atmende Firewall" — 3 Stufen basierend auf dem CBOE VIX

Anstatt einfach bei hohem VIX alle Trades zu sperren, passt das System sein Risiko asymmetrisch an — um auch in fallenden Märkten Kapital aus Short-Setups zu extrahieren.

1. Regime ALPHA (VIX < 25) — Normalbetrieb:
Longs und Shorts mit 100% Basis-Risiko. Geordneter Markt, voller Einsatz.
2. Regime DELTA (VIX 25–35) — Short-Spezialist:
Long-Trades: nur 25% Risiko (Kapitalschutz). Short-Trades: weiterhin 100% Risiko, um Panikverkäufe als Beschleuniger zu nutzen.
3. Regime OMEGA (VIX > 35) — Handelsstopp:
Beide Richtungen gesperrt (0% Risiko). Dysfunktionaler Markt — Gaps und Slippage zerstören CRVs.
🛡️

Mathematische Risiko-Kontrolle: shares = floor(aktives_Risiko / abs(Entry - SL)) — Unabhängig von der Aktien-Volatilität verbrennt das System bei einem Stop-Out punktgenau maximal dein Basis-Risiko (€).

IV. Cockpit Statusanzeige (Logik vs. Screener)

Cockpit UI und Pine Screener im Vergleich

Beide Systeme nutzen dieselbe Mathematik, werten diese aber methodisch unterschiedlich aus (Live-Tick vs. Interval-Snapshot).

COCKPIT TEXT PRIORITÄTEN (Live)

  1. 💎 AKJ ELITE: wenn isAKJSignal_l/s == true
  2. 🌟 GOLDEN: wenn trigger_golden_l/s == true
  3. 🟢 FEUER FREI: wenn trigger_feuer_l/s == true
  4. ⚠️ NO VOLUME: wenn raw_feuer_l/s == true aber Volumencheck/Zone fehlschlägt
  5. 📡 LAUERN: wenn radar_state erfüllt ist, aber WPR noch NICHT auf Spannung

ZUSÄTZE IM TEXT

  • (ALPHA/DELTA/OMEGA) = Hängt vom Risk-Mode (VIX) ab
  • (MOM) = Wenn extremes kurzfristiges Volatilitäts-Momentum vorliegt
  • [Wait: Xs] = Wenn die 30-Sekunden Anti-Spam Sperre aktiv ist
⚡

Warum kommt es zu Abweichungen (Screener +4, Cockpit LAUERN)?
Screener = Snapshot-Berechnung (z.B. alle paar Minuten am Server). Cockpit = Echtzeit-Tick im Chart.
Wenn der WPR z.B. von 80.1 (Snapshot = +4) auf 79.8 (Live-Tick = LAUERN) fällt, siehst du live was unter der Haube flackert. Der Alarm ignoriert dieses Flackern sicher durch seinen Cooldown!
